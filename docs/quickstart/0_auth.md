# Quickstart: Authentication

Step-by-step guide for getting programmatic access to the Trufo Provenance Service (TPS).

## Prerequisites

1. **Create an account** at [app.trufo.ai](https://app.trufo.ai).
2. **Set up MFA** — passkey or TOTP is required before API keys can be issued.
3. **Create an API key** at [app.trufo.ai/settings/org](https://app.trufo.ai/settings/org) under *API Keys*, picking the scope that matches your use case (see below).

## API Key Scopes

Depending on what you need, set up the corresponding API key:

- **Development / interactive access** — create a `trufo-api` key, then run `trufo login` to exchange it for an access token via the device-authorization flow.
- **Calling `/c2pa/sign` or `/test/c2pa/sign` in deployment** — create a `c2pa-sign-prod` key (for `/c2pa/sign`) or a `c2pa-sign-test` key (for `/test/c2pa/sign`).
- **Calling `tsa.trufo.ai`** — create a `tsa` key.

See [../api/api_auth.md](../api/api_auth.md) for the full scope reference.

## Step 1 — Store Your API Key

The API key is generated by Trufo at [app.trufo.ai/settings/org](https://app.trufo.ai/settings/org) under *API Keys* or within the product-specific page. Note that Trufo does NOT store a copy of your API key post-generation, so make sure to keep a copy of it securely.

To use the API key within this library, save it to a file (directly or via the CLI), or set it as an environment variable. When reading API keys, the SDK checks the environment variable first, then the file.

**Method A - CLI (recommended):**

```bash
trufo set-api-key trufo-api      <your-api-key>  # for `trufo login`
trufo set-api-key c2pa-sign-prod <your-api-key>  # for /c2pa/sign
trufo set-api-key c2pa-sign-test <your-api-key>  # for /test/c2pa/sign
trufo set-api-key tsa            <your-api-key>  # for tsa.trufo.ai
# Saved to ~/.trufo/credentials/<scope>_api_key (mode 0600)
```

**Method B - Environment variable:**

```bash
export TRUFO_API_KEY=<your-api-key>                # trufo-api
export TRUFO_C2PA_SIGN_PROD_API_KEY=<your-api-key>
export TRUFO_C2PA_SIGN_TEST_API_KEY=<your-api-key>
export TRUFO_TSA_API_KEY=<your-api-key>
```

**Load in code (automatically configured in this library):**

```python
from trufo.util.credentials import TrufoApiKey, load_api_key

api_key = load_api_key(TrufoApiKey.TRUFO_API)
assert api_key, "Set TRUFO_API_KEY or run: trufo set-api-key trufo-api <KEY>"
```

## Step 2a — Authenticate via Device Flow

Some TPS endpoints, especially the Certificate-related ones, require an *access token* (Bearer JWT) and not the API key directly. The OAuth 2.0 device authorization flow is the supported headless auth method, and requires a `trufo-api` scoped key.

**CLI (recommended):**

```bash
trufo login
# Prints something like:
#   Visit: https://app.trufo.ai/device?code=ABCD-1234
#   Enter code: ABCD-1234
# Open the URL in a browser and approve — the CLI polls automatically.
# Tokens are saved to ~/.trufo/session
```

**Python:**

```python
from trufo.api.session import TrufoSession
from trufo.util.credentials import TrufoApiKey, load_api_key, save_session

api_key = load_api_key(TrufoApiKey.TRUFO_API)

session = TrufoSession()
session.init_session(api_key=api_key)
# Prints:
#   Visit: https://app.trufo.ai/device?code=ABCD-1234
#   Enter code: ABCD-1234
# Open the URL in a browser and approve — the script polls automatically.

save_session(session)
```

Once approved, `session.access_token` and `session.refresh_token` are set.

---

## Step 2b — Load Device Flow Session in Subsequent Runs

Once `trufo login` (or `save_session()`) has run once, skip the device flow entirely:

```python
from trufo.util.credentials import load_session

session = load_session()
# Raises RuntimeError if no session is stored or tokens are missing
```

Refresh tokens are valid for **7 days** (server default). Each use of the refresh token issues a new pair, so a session stays alive as long as it is used at least once per week. After 7 days of inactivity, re-run `trufo login`.

---

## Step 3 — Make Requests

```python
data = session.make_request("/some/endpoint", {"key": "value"})
```

`TrufoSession.make_request()` handles token refresh automatically on 401. Each refresh token is single-use; the new pair is stored back on the session object.

For endpoints that accept an API key directly (e.g. the TPS signing endpoints), the key can be passed without a session.

```python
import requests
from trufo.util.credentials import TrufoApiKey, load_api_key

# /test/c2pa/sign requires a c2pa-sign-test key
test_key = load_api_key(TrufoApiKey.C2PA_SIGN_TEST)
resp = requests.post(
    "https://api.trufo.ai/test/c2pa/sign",
    json={"media_input": "...", "actions": [], "assertions": []},
    headers={"X-API-Key": test_key},
)
```
